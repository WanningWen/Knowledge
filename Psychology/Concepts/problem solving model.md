> When to use problem solving:
> one *repeatedly* talks about the same problem over period of time
> one feels like they have exhausted all their options/turned down options offered by the group
> multiple aspects of a problem pile up and needs ==help setting priorities== / breaking ==problems into manageable pieces== 
### Step 1: Define the problem
- "untangle the pile" by asking questions about the stated problem & narrowing it down
- write down each layer of the problem -> list that the individual can refer to later
- pick the ==single most pressing problem==

### Step 2: Solve the problem using “POW" 
#### P: Evaluate the ==P==ast experience
- ask and list what the individual tried in the past to solve the problem
- whether it had or had not worked (cross them off in the list)
- leave the solutions that are *somewhat* successful
#### O: Generate ==O==ptions
- ask the group for new options that are *as specific as possible*
- no option should be discounted/eliminated
#### W: Plan for "==W==hat If's"
- summarize list of options suggested
- ask client to pick a "first choice" option & underline with red market

- question: "what if this doesn't work"
- then ask to select a second option as a backup solution
- give the entire list to take home
### Step 3 set realist expectations
- remind the person to create realistic expectations of *themself* and *their family member*
